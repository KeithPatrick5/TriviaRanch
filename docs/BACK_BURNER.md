# Back Burner Features

This is the parking lot for features intentionally not built into the first Android build. This is separate from the GitHub README on purpose.

## Later, not first

- Real-time multiplayer matchmaking
- Live hosted trivia shows
- Massive public tournaments
- User-generated public question packs
- Full admin web dashboard
- AI-generated question pipeline
- Human review workflow for AI questions
- Google Play Billing subscription flow
- Revenue optimization experiments
- Private friend groups
- Clans or teams
- Voice chat
- Chat rooms
- Betting, wagering, or cash prizes
- Seasonal battle pass
- Optional rewarded ads
- iOS build
- Web landing page
- Public website leaderboard
- Push notifications
- Deep-link challenge invites
- Streak freeze purchases
- Category store
- Public custom quiz marketplace

## Why these are delayed

The first job is proving that the local Android gameplay loop is fun. Real-time multiplayer, payments, and admin tooling are expensive distractions before the core loop is validated.

## Revisit order

1. Supabase backend
2. Daily leaderboard
3. Async challenge deep links
4. Question reports and admin review
5. Push notifications
6. Google Play Billing
7. Closed testing
8. Real-time multiplayer only if async challenges retain users
